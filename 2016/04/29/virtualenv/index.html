Use pip for Installing Python Packages to Different Python Interpreter

There are 3 different python interpreters in my system. Python 2.7 is the default python interpreter linked to python in the terminal. I forget whether it was builtin the system or installed along with Xcode. Notice that Python 2.6 is also there. Meanwhile Python 3 is linked to python3.

pip and pip3 command is used to install packages for default Python and Python 3 respectively. It is set automatically when I used the python installer for Mac OS X.

Raised Problem

MySQLdb(mysql-python) is not support Python 3 and above at the moment(12062012).

At the first glance, I thought mysql-python installation failed due to serveral different Python versions existing in my system at the same time. The script setup.py was executed by python(my default python command linked to Python 2.7), while I thought it should be python3(the command used to run my Python 3.5). Furthermore, considering the ImportError of lacking module ConfigParser(import of python is case sensitive, ConfigParser is Python 2.7 builtin module and has been changed to configparser since Python 3.0), I tried to find solutions for installing package for different Python Interpreter in one system.

Use virtualenv to Sovle the Package Installing Problem in Multi-Installation Python Enviroment

http://stackoverflow.com/a/7237949
As same as pip, virtualenv is recommended for Python installation in Python Packaging User Guide.

virtualenv temp-python will make a folder temp-python in the current directory. The folder will involve the virtual enviroment for the specific Python Interpreter. The default interpreter is the one virtualenv installed with. Use virtualenv -h to check details.
